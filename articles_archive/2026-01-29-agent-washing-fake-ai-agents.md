## 衝撃の事実

**「AIエージェント」を名乗る企業の約93%が、実際にはエージェントを作っていない。**

これは誇張ではありません。業界調査によると、「AIエージェント」を謳う数千社のうち、**本当にエージェント的なシステムを構築しているのはわずか約130社**だけです。

残りは何をしているのか？

**Agent Washing（エージェント・ウォッシング）**です。

## Agent Washingとは

:::message alert
**定義:** 従来の自動化やチャットボットを「AIエージェント」とリブランディングして販売するマーケティング手法
:::

これは「グリーンウォッシング」（環境に優しいふりをする）のAI版です。

### 典型的なパターン

```
【Before】
「当社のRPAツールで業務を自動化」

【After（Agent Washing後）】
「当社のAIエージェントが自律的に業務を遂行」

→ 中身は同じ。名前だけ変えた。
```

## 本物のAIエージェントとは

### 3つの必須条件

本物のAIエージェントには、以下の能力が**すべて**必要です：

| 条件 | 説明 | 偽物との違い |
|------|------|-------------|
| **自律的判断** | 状況に応じて次のアクションを自分で決定 | 偽物：事前定義されたフローのみ |
| **ツール使用** | 外部API、DB、ファイルシステムを動的に操作 | 偽物：固定された連携のみ |
| **目標指向** | 最終目標に向かって複数ステップを計画・実行 | 偽物：単発の応答のみ |

### 具体例で比較

**偽物（チャットボット）:**
```
ユーザー: 「明日の会議をリスケして」
ボット: 「カレンダーアプリを開いて変更してください」
→ 指示を出すだけ。実行しない。
```

**本物（AIエージェント）:**
```
ユーザー: 「明日の会議をリスケして」
エージェント:
  1. カレンダーを確認（ツール使用）
  2. 参加者の空き状況を照会（API呼び出し）
  3. 最適な候補日を特定（自律判断）
  4. 参加者に確認メールを送信（アクション実行）
  5. 承認後、カレンダーを更新（目標達成）
→ 自律的に完了まで実行。
```

## なぜAgent Washingが蔓延しているのか

### 1. 投資マネーの流れ

```

           ↓
「エージェント」と名乗らないと資金調達できない
           ↓
製品の実態に関係なく「エージェント」を名乗る
```

### 2. バズワード効果

「AIエージェント」という言葉は、2025-2026年で最もホットなバズワードの一つ。名乗るだけで注目を集められる。

### 3. 定義の曖昧さ

「AIエージェント」に業界標準の定義がない。だから誰でも名乗れる。

## 本物を見分ける5つの質問

製品やサービスを評価するとき、以下を確認してください：

### Q1: 「予期しない状況にどう対応しますか？」

- **本物:** 「LLMが状況を判断し、適切なアクションを選択します」
- **偽物:** 「エラーハンドリングで定義済みの処理に分岐します」

### Q2: 「ワークフローはどこで定義されていますか？」

- **本物:** 「エージェントが目標に基づいて動的に計画します」
- **偽物:** 「管理画面でフローを設定します」（= 従来のRPA）

### Q3: 「新しいツールの追加は？」

- **本物:** 「MCP/A2Aプロトコルで接続すれば自動認識します」
- **偽物:** 「開発チームがインテグレーションを実装します」

### Q4: 「人間の介入なしで完了できるタスクは？」

- **本物:** 具体的な複数ステップのタスクを説明できる
- **偽物:** 「人間の承認が必要です」「確認後に実行します」

### Q5: 「失敗したらどうなりますか？」

- **本物:** 「別のアプローチを試みます」「理由を説明して代替案を提示します」
- **偽物:** 「エラーを返します」「サポートに連絡してください」
